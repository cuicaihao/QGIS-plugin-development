                                                                                                                                                          Anakaine Ananth (Front end developer,hi a year ago edited

                                                                                                                                                          This is rarely a god idea. If you are concerned about data security for access, use an appropriately constructed database provider, with appropriate access levels. If the data needs to be encrypted at rest, either use the database with appropriate permissions and an encrypted-at-rest method compatible with your DB, or use an OS controlled encrypted filing system rather than attempting to encrypt the GIS data itself. By encrypting the GIS data in your GIS application or local filestore based on GIS filetype you will be creating significant processing bottlenecks and potential recall issues in the future, not to mention you will also be on the wrong side of GIS data storage standards.
